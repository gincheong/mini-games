# 🏠 집안 물건 찾기 앱 — Claude Code 프롬프트 시리즈

## ✅ Phase 1. 프로젝트 세팅

```
React + Vite로 새 프로젝트를 만들어줘.

기술 스택:
- React 18 + Vite
- react-konva (캔버스 드로잉/인터랙션)
- zustand (전역 상태 관리)
- Tailwind CSS (UI 스타일링)

아래 구조로 폴더와 파일을 만들어줘:

src/
  components/
    canvas/         # 평면도 캔버스 관련
    furniture/      # 가구 오브젝트 관련
    drawer/         # 서랍 관련
    ui/             # 공통 UI 컴포넌트
  store/
    useRoomStore.js    # 방/평면도 상태
    useFurnitureStore.js  # 가구/서랍/아이템 상태
  App.jsx
  main.jsx

그리고 아래 데이터 구조를 기반으로 zustand store 초기 코드를 작성해줘:

// Room
{
  id: string,
  name: string,
  walls: [ {x, y}[] ]  // 폴리곤 꼭짓점 배열
}

// Furniture
{
  id: string,
  roomId: string,
  label: string,       // 예: "서랍장", "옷장"
  type: string,        // 'dresser' | 'wardrobe' | 'shelf' | 'custom'
  x: number,
  y: number,
  width: number,
  height: number,
  rotation: number,
  drawers: Drawer[]
}

// Drawer
{
  id: string,
  furnitureId: string,
  label: string,       // 예: "1단", "2단"
  order: number,
  items: Item[]
}

// Item
{
  id: string,
  drawerId: string,
  name: string,
  memo: string,
  tags: string[]
}

패키지 설치 명령어도 알려줘.
```

## ✅ Phase 2. 캔버스 기본 레이아웃

```
이전에 만든 프로젝트에서 메인 레이아웃을 만들어줘.

화면은 좌우 2분할:
- 왼쪽: 도구 패널 (사이드바, 너비 240px)
- 오른쪽: Konva Stage (캔버스 영역, 나머지 전체)

사이드바에는 아래 섹션을 만들어줘:
1. 방 목록 (방 추가 버튼 포함)
2. 가구 추가 버튼들 (서랍장, 옷장, 선반, 커스텀)
3. 현재 선택된 오브젝트 정보 표시 영역

캔버스(Konva Stage)에는:
- 격자 배경 (grid, 20px 간격)
- 현재 선택된 방의 Layer만 표시
- 빈 상태일 때 "방을 먼저 추가하세요" 안내 텍스트

디자인은 깔끔한 다크 테마로 해줘.
Tailwind로 스타일링하고, 전체적으로 집 관리 앱다운 실용적인 느낌으로.
```

## ✅ Phase 3. 방 그리기 기능

```
캔버스에서 방(평면도)을 그리는 기능을 구현해줘.

동작 방식:
- 사이드바에서 "방 추가" 버튼 클릭 → 드로잉 모드 진입
- 드로잉 모드에서 캔버스를 클릭할 때마다 꼭짓점 추가
- 꼭짓점들을 선으로 연결해서 실시간으로 보여줌
- 첫 번째 꼭짓점 근처에 마우스를 가져가면 하이라이트 표시
- 첫 번째 꼭짓점 클릭 시 폴리곤 완성 → 일반 모드로 복귀
- ESC 키로 드로잉 취소

완성된 방:
- 반투명 채우기 색상으로 표시
- 방 이름 텍스트를 중앙에 표시
- 클릭하면 선택 상태 (선택 시 테두리 강조)
- 선택 상태에서 Delete 키 → 방 삭제 확인 모달

useRoomStore에 walls 데이터를 저장하고,
현재 드로잉 상태는 로컬 state로 관리해줘.
```

## ✅ Phase 4. 가구 배치 기능

```
캔버스에 가구 오브젝트를 배치하는 기능을 구현해줘.

가구 추가:
- 사이드바의 가구 버튼 클릭 → 현재 선택된 방 중앙에 가구 오브젝트 생성
- 방이 선택되지 않은 상태면 "방을 먼저 선택하세요" 토스트 메시지

가구 오브젝트 (Konva Group):
- 배경 사각형 (종류별로 색상 다르게)
- 가구 라벨 텍스트
- 가구 종류 아이콘 (이모지로 간단하게: 서랍장🗄️, 옷장🚪, 선반📚)

인터랙션:
- 드래그로 위치 이동 (방 영역 밖으로 못 나가게 제한)
- 클릭으로 선택 → 선택된 가구는 테두리 강조 + 사이드바에 정보 표시
- 선택된 상태에서 모서리 핸들로 크기 조절
- 선택된 상태에서 회전 핸들로 회전
- 더블클릭 → 가구 상세 패널 열기

useFurnitureStore에 가구 데이터 저장.
```

## ✅ Phase 5. 가구 상세 패널 (서랍/아이템 관리)

```
가구를 더블클릭했을 때 열리는 상세 패널을 만들어줘.

패널 형태: 오른쪽에서 슬라이드인 되는 사이드 패널 (너비 360px)

패널 내용:
1. 상단: 가구 이름 편집 (인풋 클릭하면 바로 수정 가능)

2. 서랍 목록:
   - 각 서랍은 카드 형태로 표시
   - 서랍 이름 편집 가능 (예: "1단 서랍", "2단 서랍")
   - 서랍 카드 클릭하면 해당 서랍의 아이템 목록 펼쳐짐 (아코디언)
   - 서랍 추가 버튼 (하단에 "+ 서랍 추가")
   - 서랍 삭제 버튼 (각 카드에 X 버튼)
   - 서랍 순서 변경 (드래그 핸들로 순서 재정렬)

3. 아이템 관리 (서랍 펼쳐졌을 때):
   - 아이템 목록 (이름 + 메모 + 태그)
   - 아이템 추가: 이름 입력 후 엔터 또는 추가 버튼
   - 아이템 클릭하면 메모/태그 편집 가능
   - 아이템 삭제 버튼

useFurnitureStore의 drawers, items 업데이트 로직 포함해줘.
```

## ✅ Phase 6. 검색 기능

```
전체 아이템을 검색하는 기능을 추가해줘.

검색창:
- 상단 헤더에 검색 인풋 배치
- 실시간 검색 (타이핑하는 즉시 결과 표시)

검색 결과:
- 아이템 이름, 메모, 태그 전체에서 검색
- 결과는 드롭다운 또는 별도 결과 패널로 표시
- 각 결과 카드에: 아이템명 / 위치 경로 표시 (예: "침실 → 서랍장 → 2단 서랍")
- 결과 클릭 시: 해당 방으로 이동 + 해당 가구 선택 + 상세 패널 오픈

검색 결과 없을 때 "찾을 수 없어요 🤷" 빈 상태 UI도 만들어줘.
```

## ✅ Phase 7. 데이터 저장 (로컬스토리지)

```
앱 데이터를 localStorage에 자동 저장/불러오기 하는 기능을 추가해줘.

요구사항:
- zustand persist 미들웨어 사용
- 저장 키: 'home-finder-data'
- rooms, furniture(drawers, items 포함) 전체 저장
- 앱 시작 시 자동으로 저장된 데이터 복원

추가로:
- 헤더에 "내보내기" 버튼 → 전체 데이터를 JSON 파일로 다운로드
- 헤더에 "가져오기" 버튼 → JSON 파일 업로드해서 데이터 복원
- 데이터 초기화 버튼 (확인 모달 포함)
```
